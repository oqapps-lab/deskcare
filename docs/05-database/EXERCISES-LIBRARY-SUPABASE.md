# DeskCare — Библиотека упражнений (атомы) для Supabase

**Дата:** 2026-04-25
**Источник:** `MVP-EXERCISES-LIST.md` (промты Seedance) + `deskcare/docs/05-database/DATABASE-SCHEMA.md` (структура БД)
**Назначение:** Каталог атомов — единиц контента (бесшовно зацикленных видео) для таблицы `exercises` в Supabase. Один атом = одна строка БД, может относиться к нескольким зонам тела через M:N связь `exercise_body_zones`.
**Связанные документы:**
- `deskcare/docs/05-database/ROUTINES-CATALOG.md` — формулы рутин по категориям (atom × reps).
- `deskcare/docs/07-development/ROUTINE-PLAYER.md` — логика React Native плеера.

**Статус видео:** все 64 атома сняты ✅. Eye-секция (8 атомов) реализована как видео-loop, не Lottie.

---

## Легенда

| Знак | Значение |
|------|----------|
| ✅ | Видео снято, есть `video_url` в Supabase Storage |
| 🔁 | Кросс-категорийная ссылка — атом уже существует в БД, добавлен ещё один M:N в `exercise_body_zones` |

**Сложность (difficulty):**
- `1` — easy (любой пользователь, нет противопоказаний)
- `2` — medium (требует базовой мобильности)
- `3` — hard (только для фазы поддержания / опытных)

**Колонки таблиц:**
- **Время** — длительность одного бесшовного проигрыша атома, в секундах. Это `exercises.duration_seconds`.
- **Кол-во повторов в атоме** — сколько «реальных повторений упражнения» уже включено внутри одного проигрыша. Например, N7 (Круги локтями) внутри уже делает 3 круга вперёд + 3 назад + возврат — это считается «1 повтор атома». В рутине этот атом можно проиграть `N7 × 2 = 20 сек` для удвоения работы, и т.д.
- **Атомы стыкуются бесшовно** — последний кадр совпадает со стартовым, поэтому `atom × N` воспроизводится без видимого скачка.

## Маппинг на схему Supabase

Каждый атом в этом документе соответствует одной строке в таблице `exercises`:

```sql
exercises (
  id              uuid pk,
  slug            text unique,         -- 'neck-lateral-tilt' (URL-friendly)
  code            text unique,         -- 'N1' (внутренний код, для удобства)
  title           text,                -- "Наклоны головы в стороны"
  title_en        text,                -- "Lateral Neck Tilt"
  description     text,                -- короткое UI-описание (см. ниже)
  video_url       text,                -- ссылка в exercise-videos бакете
  thumbnail_url   text,
  duration_seconds smallint default 5, -- длина loop-клипа
  difficulty      smallint,            -- 1/2/3
  exercise_type   text,                -- stretch | mobility | nerve_glide | tendon_glide | breathing | strength
  is_premium      boolean default false,
  target_muscles  text[],
  contraindications text[],
  cautions        text                 -- предупреждение в UI (для W8, S-серии)
)

-- Связь зон через exercise_body_zones (M:N)
-- Один атом N5 (Круги плечами) → две записи: (N5, neck), (N5, full_body)
exercise_body_zones (
  exercise_id  uuid fk,
  body_zone_id uuid fk,
  is_primary   boolean   -- true = основная категория (Home), false = появляется в Library по фильтру
)
```

**Body zones (slugs):** `neck`, `back`, `wrists`, `full_body`, `sciatica`, `eyes`.

**Принцип кросс-категории:** ниже в каждой секции есть блок «Кросс-категорийные атомы» — упражнения, продублированные из других секций под новыми кодами (N11+, B12+, F12+ и т.д.). В Supabase это **не новые строки** — это вторая запись в `exercise_body_zones` для уже существующего атома. Новые коды нужны только как маркер «этот атом фильтруется в эту категорию» в Library и используется в рутинах этой зоны.

# 9. Как формируются рутины (краткий справочник для seed-скрипта)

Рутина — это упорядоченная последовательность пар `(exercise_code, reps)`. Длительность рутины:

```
total_duration = Σ (atom.duration_seconds × reps) + Σ rest_seconds
```

Формула в коротком виде: `N1×5 + N2×5 + N3×4 + N10×2`. Это значит:
- N1 (6 сек) проигрывается 5 раз бесшовно подряд → 30 сек
- N2 (6 сек) × 5 = 30 сек
- N3 (7 сек) × 4 = 28 сек
- N10 (9 сек) × 2 = 18 сек
- **Итого:** 106 сек ≈ 1:46 → подходит под бюджет «2-минутная рутина»

**Гайдлайны при подборе reps:**
- Динамика/мобилизация (mobility, strength) — 2–4 повтора, чтобы успеть «прогреть».
- Удержания/растяжки (stretch) — 1–2 повтора, удержание уже встроено в атом.
- Нерв-глайды — 1–2 повтора, перебарщивать нельзя (раздражает нерв).
- Дыхание — 1–2 цикла (он уже длинный, S6 = 14 сек).
- Атом с 2 сторонами (по 1 на сторону) — нечётные reps асимметричны, лучше чётное число.

**Конкретные рутины с расчётом длительностей** — см. `deskcare/docs/05-database/ROUTINES-CATALOG.md`.
